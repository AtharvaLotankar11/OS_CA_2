# Edge Computing: Vision and Challenges

[![Edge Computing Image](img.gif)](http://dx.doi.org/10.1109/JIOT.2016.2579198)

This repository contains a study and analysis of Edge Computing, as detailed in the paper "Edge Computing: Vision and Challenges" by Weisong Shi, Jie Cao, Quan Zhang, Youhuizi Li, and Lanyu Xu. [cite: 886, 887, 888, 889]

## Overview

The proliferation of Internet of Things (IoT) and the increasing demand for rich cloud services have led to the development of a new computing paradigm: Edge Computing. [cite: 886, 887, 888, 889, 890, 891, 892, 893, 894, 895, 896, 897, 898, 899, 900, 901, 902, 903, 904, 905, 906, 907, 908] Edge Computing addresses the limitations of traditional cloud computing by processing data at the edge of the network, closer to the data source. [cite: 886, 887, 888, 889, 890, 891, 892, 893, 894, 895, 896, 897, 898, 899, 900, 901, 902, 903, 904, 905, 906, 907, 908]

### Key Concepts

* **Edge Computing Definition:** Computing performed at the edge of the network, between data sources and cloud data centers. [cite: 940, 941, 942, 943, 944, 945, 946, 947, 948, 949, 950, 951, 952, 953, 954, 955, 956, 957, 958, 959, 960, 961, 962, 963, 964]
   
* **Benefits of Edge Computing:**
   * Reduced response time [cite: 886, 887, 888, 889, 947, 948, 949, 950, 951, 952, 953, 954, 955, 956, 957, 958, 959, 960, 961, 962, 963, 964]
   * Lower energy consumption [cite: 886, 887, 888, 889, 947, 948, 949, 950, 951, 952, 953, 954, 955, 956, 957, 958, 959, 960, 961, 962, 963, 964]
   * Decreased bandwidth usage [cite: 886, 887, 888, 889, 947, 948, 949, 950, 951, 952, 953, 954, 955, 956, 957, 958, 959, 960, 961, 962, 963, 964]
   * Enhanced data privacy and security [cite: 886, 887, 888, 889, 947, 948, 949, 950, 951, 952, 953, 954, 955, 956, 957, 958, 959, 960, 961, 962, 963, 964]

### Case Studies

The paper explores several case studies to illustrate the applications and potential of Edge Computing:

* **Cloud Offloading:** Offloading computing tasks from the cloud to edge nodes to reduce latency and improve user experience. [cite: 957, 958, 959, 960, 961, 962, 963, 964, 965, 966, 967, 968, 969, 970, 971, 972, 973, 974, 975, 976, 977, 978, 979, 980, 981]
   
* **Video Analytics:** Utilizing edge computing for real-time video analysis, enhancing applications like surveillance and search operations. [cite: 982, 983, 984, 985, 986, 987, 988, 989, 990, 991, 992, 993, 994, 995, 996, 997, 998, 999, 1000, 1001, 1002, 1003]
   
* **Smart Home:** Implementing edge computing to manage and process data from various smart devices within a home environment. [cite: 982, 983, 984, 985, 986, 987, 988, 989, 990, 991, 992, 993, 994, 995, 996, 997, 998, 999, 1000, 1001, 1002, 1003]
   
* **Smart City:** Deploying edge computing on a city-wide scale to handle large data quantities, reduce latency for critical applications, and leverage location awareness. [cite: 1012, 1013, 1014, 1015, 1016, 1017, 1018, 1019, 1020, 1021, 1022, 1023, 1024, 1025, 1026, 1027]
   
* **Collaborative Edge:** Connecting edges of multiple stakeholders to enable data sharing and cooperation across different domains, particularly beneficial in applications like connected healthcare. [cite: 1012, 1013, 1014, 1015, 1016, 1017, 1018, 1019, 1020, 1021, 1022, 1023, 1024, 1025, 1026, 1027]

### Challenges and Opportunities

The paper identifies several challenges and opportunities in the field of Edge Computing:

* **Programmability:** Developing efficient ways to program and deploy applications across heterogeneous edge devices. [cite: 1061, 1062, 1063, 1064, 1065, 1066, 1067, 1068, 1069, 1070, 1071, 1072, 1073, 1074, 1075, 1076, 1077, 1078, 1079, 1080, 1081, 1082, 1083, 1084, 1085, 1086]
   
* **Naming:** Creating robust and scalable naming mechanisms for the large number of devices and services in edge networks. [cite: 1061, 1062, 1063, 1064, 1065, 1066, 1067, 1068, 1069, 1070, 1071, 1072, 1073, 1074, 1075, 1076, 1077, 1078, 1079, 1080, 1081, 1082, 1083, 1084, 1085, 1086]
   
* **Data Abstraction:** Effectively managing and abstracting data from diverse sources in edge environments. * **Service Management:** Ensuring reliability, differentiation, extensibility, and isolation of services at the edge.
